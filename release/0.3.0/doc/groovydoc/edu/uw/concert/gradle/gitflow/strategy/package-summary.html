Package edu.uw.concert.gradle.gitflow.strategy

Package edu.uw.concert.gradle.gitflow.strategy Description

Provides the strategies for inferring the version of the project to which the plugin is applied.

The entry point to this package is the interface Strategy.

There is exactly one class for each type of branch defined in the Gitflow Branching Model that provides a strategy to infer the version of the project. Furthermore, there is DetachedHeadStrategy that applies if the current commit is a detached head. AbstractStrategy is the base class of these strategies and provides common logic like logging.

  • Interface Summary 
    Interface Description
    Strategy The interface for all strategies used to infer the version.
  • Class Summary 
    Class Description
    AbstractStrategy The base class of all strategies used to infer the version.
    BranchDevelopStrategy The strategy to use when Gitflow's develop branch is the current branch.
    BranchFeatureStrategy The strategy to use when one of Gitflow's feature branches is the current branch.
    BranchHotfixStrategy The strategy to use when one of Gitflow's hotfix branches is the current branch.
    BranchPreReleaseStrategy The strategy to use when one of Gitflow's pre-release / next release branches is the current branch.
    BranchReleaseStrategy The strategy to use when Gitflow's production release branch is the current branch.
    BranchSupportStrategy The strategy to use when one of Gitflow's support branches is the current branch.
    DetachedHeadStrategy The strategy to use when the current head is a detached head.
